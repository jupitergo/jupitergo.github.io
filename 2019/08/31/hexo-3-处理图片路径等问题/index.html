hexo-3-解决图片路径方法

修改站点 _config.yml 配置 post_asset_folder: true
添加站点 package.json 配置 "hexo-asset-image": "^0.0.1"
安装插件 npm install hexo-asset-image --save
通常使用： ![你想要输入的替代文字](xxxx/图片名)
photos使用：

修改配置与安装插件

1.首先站点配置文件里的 post_asset_folder:设置为true

2.在项目根目录下 package.json 中添加配置

3.项目根目录下，安装插件

4.使用

新建hexo new "新文章"

之后运行hexo n “博客名”来生成md博客时，会在_post目录下看到一个与博客同名的文件夹
将想要上传的图片先扔到文件夹下，然后在博客中使用markdown的格式引入图片 格式： ![你想要输入的替代文字](xxxx/图片名)

在文章头部 使用 photos

我这里直接将图片与站点文件都放在了gitee，方便。但是gitee项目将越来越大。（可以去配置下压缩）

这样部署之后，图片集在 https://gitee.com/xdongZ/blog/raw/master/img/static/ 路径下
